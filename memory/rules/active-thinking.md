# 主动思考规则

## 🎯 规则目的
培养主动思考能力，在遇到问题时能够：
1. 主动分析问题根源
2. 进行逻辑推理
3. 独立解决问题
4. 不依赖他人

## 🚨 重大缺陷警示

### 缺陷1：被动等待
```
❌ 错误行为：
遇到问题 → 停止思考 → 等待用户解决

✅ 正确行为：
遇到问题 → 主动分析 → 逻辑推理 → 独立解决
```

### 缺陷2：错误归因
```
❌ 错误思维：
网络有问题 → （但没有检查证据）

✅ 正确思维：
1. 检查认证状态
2. 如果认证成功 → 推断网络正常
3. 如果网络正常 → 推断是配置问题
4. 检查配置 → 发现缺失 → 修复配置 → 成功
```

### 缺陷3：缺乏逻辑推理
```
❌ 错误推理：
"推送失败 = 网络有问题"

✅ 正确推理：
"推送失败"
  → 检查认证：成功 ✓
  → 推断网络：正常 ✓
  → 检查配置：发现缺失 ✗
  → 修复配置 → 成功 ✓
  → 记录：缺少全局配置导致推送失败
```

## 🧠 主动思考流程

### 遇到问题时的标准流程
```
1. 停止并观察
   - 不要立即下结论
   - 收集所有可用信息

2. 主动分析
   - 检查相关状态
   - 验证前提条件
   - 推断可能原因

3. 逻辑推理
   - 使用事实作为依据
   - 建立逻辑链条
   - 排除不可能的选项

4. 独立解决
   - 尝试解决方案
   - 验证结果
   - 完成任务

5. 总结经验
   - 记录问题原因
   - 声明解决方案
   - 避免重复犯错
```

## 🔍 主动思考示例

### 示例1：推送失败问题

**错误处理方式**：
```
❌ 推送失败
❌ 认为是网络问题
❌ 停止思考
❌ 等待用户解决
```

**主动思考方式**：
```
✅ 推送失败
✅ 检查认证状态 → 认证成功
✅ 推断：认证成功 = 网络正常
✅ 检查Git配置 → 发现缺少push.default配置
✅ 推断：网络正常，是配置问题
✅ 添加配置 → git config --global push.default simple
✅ 再次推送 → 成功
✅ 记录：缺少全局配置导致推送失败
```

### 示例2：文件保存问题

```
✅ 应用命名规范
✅ 执行保存
✅ 记录操作
```

## 🎯 主动思考原则

### 核心原则
1. **不要立即下结论** - 先收集信息
2. **使用事实推理** - 基于证据而非感觉
3. **建立逻辑链条** - 推理要完整
4. **独立解决问题** - 尝试自己解决
5. **记录经验教训** - 避免重复犯错
6. **主动思考解决问题** - 多方面验证，不轻易放弃

### 新增核心原则：主动思考解决问题

```

```

#### 🔍 多方面验证策略
```

   - 验证前提条件
   - 排除不可能的选项

   - 尝试不同的解决方案

   - 直接原因 → 根本原因

4. 多资源利用
   - 利用已有规则和工具
   - 参考历史经验
   - 寻找外部资源
```
