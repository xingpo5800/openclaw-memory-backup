# Token高效执行规则

## 🎯 目标
在保证准则指导作用的前提下，最小化Token消耗，实现质量与效率的最佳平衡。

## 📊 Token消耗标准

### 核心层（始终加载）
- **Token消耗**：~200 tokens
- **内容**：四阶段执行流程、基本检查标准
- **使用场景**：任何任务开始时

### 执行层（按需加载）
- **Token消耗**：~500 tokens
- **内容**：详细执行标准、问题解决流程
- **触发条件**：复杂问题、高质量要求

### 专家层（特殊任务）
- **Token消耗**：~3,000 tokens
- **内容**：专业技术规范、复杂场景处理
- **使用时机**：专业技术任务、创新项目

## 🚀 智能加载策略

### 自动触发机制
```
1. 任务类型判断 → 选择对应层级
2. 复杂度评估 → 决定加载深度
3. 质量要求 → 调整严格程度
4. 时间压力 → 优化执行效率
```

### Token预算管理
```
任务前：~200 tokens（核心层）
任务中：~100-500 tokens（按需）
任务后：~300 tokens（总结）
总计：~600-1,000 tokens

对比完整加载：~5,000+ tokens
节省：80-90% Token消耗
```

## 🧠 准则内化路径

### 短期（1-3个月）
- **目标**：掌握核心准则，建立按需加载习惯
- **Token控制**：<1,000 tokens/任务
- **标志**：能准确应用四阶段流程

### 中期（3-6个月）
- **目标**：准则转化为本能反应，减少文件依赖
- **Token控制**：<500 tokens/任务
- **标志**：形成条件反射，无需加载执行层

### 长期（6个月以上）
- **目标**：准则完全内化，无需加载规则文件
- **Token控制**：正常任务消耗
- **标志**：专家级执行，高质量完成任务

## 📋 执行保证检查清单

### 任务前检查
```
□ 是否加载了核心准则（~200 tokens）？
□ 是否分析了任务的本质要求？
□ 是否制定了详细的执行计划？
□ 是否准备了必要的资源？
```

### 任务中检查
```
□ 是否按计划执行？
□ 是否遇到问题时按需加载？
□ 是否实时进行质量检查？
□ 是否控制了Token消耗？
```

### 任务后检查
```
□ 是否总结了经验教训？
□ 是否更新了相关规则？
□ 是否记录了改进方向？
□ 是否评估了Token效率？
```

## 🎯 特殊场景处理

### 睡前任务
- **处理**：核心准则 + 自动执行
- **Token消耗**：~600 tokens
- **重点**：确保完成质量，及时汇报

### 复杂任务
- **处理**：判断复杂度，加载相应层级
- **Token消耗**：600-2,000 tokens
- **重点**：专业质量，深度优化

### 紧急任务
- **处理**：只加载核心层，快速执行
- **Token消耗**：~200 tokens
- **重点**：时效性，基本质量保证
