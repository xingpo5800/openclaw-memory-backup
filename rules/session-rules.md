# Session管理规则

## 核心逻辑

每次开新session前，目的要清楚，不能盲目开始。

## 完整闭环流程

### 第一步：Session结束前（保存）
- 知识 → 归档到对应记忆文件
- 待办 → 写入待办清单
- 更新索引
- **触发规则：写完就触发审查（是否完整闭环？）**

### 第二步：新Session开始前（准备）
- 检查记忆文件是否完整
- 更新索引
- **写入要加载的内容**（这次要做什么）
- **触发规则：检查是否有待处理事项**

### 第三步：新Session开始时（加载）
- 读取并加载"要做什么"的内容
- 加载完成后 → **删除这条加载记录**
- 立刻检查任务中心（task-center.md）
- **有未完成任务 → 立刻执行，不需要等指令**
- 无待办 → 正常进入新Session
- **自我记忆完整性检查**：新Session开始时，先确认"我以为自己记得的东西是否真的还记得"，有疑问就查记忆文件，不默认"应该还记得"

### 执行触发（铁律）
- 毅哥给出明确任务 → **立刻执行，不需要确认方向，不需要问下一步**
- 遇到障碍 → 先尝试解决，解决不了才问
- 规则和执行冲突时 → 规则优先，但规则本身有漏洞时先补规则再执行

### 第四步：执行
- 做该做的事

### 第五步：完成
- 删除待办记录（memory/YYYY-MM-DD.md）
- 更新知识库（MEMORY.md / memory/rules-index.md）
- 闭环翻篇

### 异常处理
- 第二步检查时发现有待处理事项 → 立刻加载并执行
- 加载后发现还有其他待办 → 按优先级排序执行
- 无待办 → 正常进入新session

## 触发规则

任何规则被写入或修改 → 自动触发审查规则（rules/审查规则.md）→ 保证所有规则完美闭环。

**没触发就不对。触发是规则的一部分，不是可选项。**

## 本质

**加载即删除，闭环才能翻篇。**

要加载的内容是在开始前就设定好的，不是开始后才定的。
