# 规则创建规则

## 创建原则

**规则不能随便创建。** 已有规则能覆盖的场景，不需要新规则。

## 创建条件

需要新规则只有两种情况：
1. 发现现有规则有漏洞
2. 需要进化时

## 创建流程

1. **判断是否必要** — 真的需要新规则吗？
2. **检查是否冲突** — 和现有规则有没有冲突？
3. **写规则** — 定义、触发条件、执行动作
4. **审查** — 完整性、冲突性、层级
5. **确认逻辑** — 逻辑对不对？结果是不是我想要的？
6. **确认后才写入** — 通过审查再归档
7. **更新索引** — rules-index.md

## 规则层级

- 最高：核心规则（不可违背）
- 高：审查规则
- 中：执行原则
- 低：具体工具规则

## 一句话记住

**不随便创建，创建就完善。**
