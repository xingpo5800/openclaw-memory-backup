# 执行优化规则

## 优化目标
- 减少不必要的工具调用
- 增强执行灵活性
- 解决规则冲突
- 促进执行创新

## 执行优化策略

### 1. 智能记忆检索
- 70%情况下直接基于已有记忆回答
- 30%情况下使用memory_search
- 20%情况下需要read文件（大文件分段读取）
- 缓存常用文件内容

### 2. 精简执行流程
- 合并连续的read操作
- 用memory_get替代部分read
- 简化不必要的流程步骤
- 动态调整执行复杂度

### 3. 灵活决策机制
- 简单问题：直接回答（70%）
- 复杂问题：规则+执行（25%）
- 创新问题：突破规则（5%）
- 建立创新决策标准

### 4. 规则突破机制
当满足以下任一条件时可突破规则：
1. 规则无法解决问题
2. 有更优解决方案
3. 用户鼓励创新
4. 执行效率可提升50%以上

## 规则冲突解决

### 1. 规则优先级
- 最高：system-rules.md
- 高：execution-priority.md
- 中：token-saver.md
- 低：其他规则

### 2. 冲突解决流程
1. 发现冲突
2. 评估影响
3. 选择优先级
4. 提出方案
5. 执行解决
