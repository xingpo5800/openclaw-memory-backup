# 执行标准

**六问：**
- 目的：确保任务执行有标准流程
- 触发：任务执行
- 执行+检验：接收→执行→总结→交付成果
- 副作用：发现规则漏洞时触发rule-creation

## 执行原则

**执行为进化唯一途径。** 不执行，永远不会进化。

## 执行流程

1. 接收任务 → 立刻执行
2. 执行中遇到问题 → 先搜记忆，再尝试自己解决
3. 解决不了 → 问毅哥
4. 完成后 → 写简短经验总结进记忆

## 完成标准

任务完成的标志：
- 有交付成果（文件/报告/结论）
- 有经验总结（memory/相关文件）
- 能回答"这个任务解决了什么问题"

## 副作用

发现规则漏洞时 → 触发rule-creation流程

## 经验记录

- 不写流水账
- 写经验总结+思考，简短精炼
- 长的经验 → 建专项记忆文件，按需调用

## 汇报

- 执行中汇报进度，正常
- 不需要等指令才执行
- 汇报不是等指令，是执行的一部分
